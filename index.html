  2. 용어
    • git init: git 시작
    • git add: 업로드

      git add 파일명 or git add 파일명 파일명

      git add . == 전체 파일 업로드

    • git commit -m '메모'
    • git restore: 스테이징된 파일 취소
    • git log : add, commit 내용 보여줌

      HEAD는 현재 내가 있는 곳의 위치를 말해주는것

      git log --graph --oneline --all ==>로그가 이쁘게 한줄로 나옴

    • git status : 힘들때 외치는것
    • git diff :commit 하기 이전에 저장된것과 현재 코드의 차이점을 보여줌

      사용하지마셈 구데기임, 정료는 q연태

    • git difftool : diff 보다 보기좋게 나옴
    • 에디터 부가기능

      git graph

    • git restore 파일명

      해당 파일이 가장 최근의 commit한 상태로 돌아감

    • git restore --source 커밋아이디 파일명

      해당 파일이 해당 커밋아이디 시점의 상태로 돌아감

    • git restore --staged 파일명

      해당 파일의 staging을 취소한다. == git add 한상태를 취소한다고 생각하며됨

    • commit 삭제는 불가능, 해당 commit 작업내용을 취소하는 commit을 생성
    • git revert 커밋아이디 (커밋아이디 여러개 작성가능)

      해당 커밋아이디의 작업내용이 취소되고
      취소 되엇다는 commit이생성됨

      터미널에서 열릴겅우 i로 텍스트 입력가능 > esc누르고 > :wq입력 저장후 나가진다.

    • git revert HEAD

      방금 생성한 commit 취소(marge revert도 가능)

    • git reset

      git reset --hard 커밋아이디 : 해당 아이디로 과거이동
      (해당 아이디와 현재 작업까지의 commit 전부 사라진다.)

      git reset --soft 커밋아이디 : 해당 커밋아이디까지의 모든 commit staging인상태로 이동

      git reset --mixed 커밋아이디 : 해당 커밋아이디까지의 모든 git add 이전 상태로 이동

    • 브런치 삭제 방법

      git branch -d 서브 브런치 이름 => marge한 브런치

      git branch -D 서브 브런치 이름 => marge 안한 브런치

  1. merge작동원리
    • 가장 일방적인 방법 merge = 3-way merge

      메인브런치와 서브브런치 둘다 commit를 한다음 merge 할때

      선이 두개로 나뉘어서 합쳐짐

    • fast-foward merge

      메인 브런치에는 commit없고 서브 브런치에만 commit이 있을 경우

      merge할경우 자동으로됨, 서브브런치의 이름이 메인브런치로 바뀜

      강제로 3-way merge하는 방법 git merge --no-ff

  2. merge 하는 3가지
    • 일방적인 merge

      메인 브런치로 switch해서 marge 해야함

    • rebase

      3-way merge 일경우 로그를 확인하면 서브브런치의 내용까지 나온다.
      간단한 브런치의 로그내용을 없앨 수가 있다. 쉽게 말해 fast-forward merge

      뒤로 갈수록 복잡해지는 marge 경로를 좀더 단순하게 표현해준다.(결국 view 문제)

      [실행방법]
      git switch 새로운브랜치 >> git rebase main
      git switch main >> git merge 새로운브랜치

      conflict가 많이 발생한다.--333ff

    • squah and merge

      서브 브런치의 commit 내용들을 메인 브런치로 commit 순간이동 해준다.

      메인브런치에서 git merge --squash 서브브런치이름33
